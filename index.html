Projects

#EdPain

http://edpain.digitalharborfoundation.org

This is a web application to collect pain points in education. At Digital Harbor Foundation, we wanted three outcomes:

  1. Increased empathy in conversations about education (by informing people of the pains parties in the educational process experience)
  2. Ability to pinpoint clusters of similar pains
  3. Find individuals more likely to be early adopters for relevant solutions

This was an MVP and was in production two weeks after ideation.

Technologies Used

  • HTML5 - For custom typefaces and responsiveness
  • Socket.io - For live updates
  • Mongodb - For data persistance
  • Redis - For session and socket.io scalability
  • Node.js - Express-hosted on Heroku

Reinvent the Report Card

http://reportCardReinvention.herokuapp.com

I made this 'live' mockup as practice for graphing, SVG, HTML5, and CSS3 practice in November 2011. I found an award-winning design and after obtaining permission from the designer, coded the mockup over episodes of TNG on Netflix. I was really playing with the limits of SVG, so I used it in many places a png or gif would have been better. It's worth noting that the typeface choices on the live mockup are mine not Polly's and that they are terrible.

Technologies Used

  • CSS - For a complex UI
  • SVG - For the graphs
  • jQuery SVG - For building complex SVGs– I should have used D3.js
